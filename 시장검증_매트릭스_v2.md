# 「어떤 서비스를 만들고, 어떻게 성공시키나」 마스터 매트릭스 v2
### 발상 → 설계 → 검증 → 성장까지, 하나로 꿰는 평가·설계 프레임

> YC·Paul Graham·Andreessen·Sequoia·Thiel·Bill Gross·Sean Ellis·Ash Maurya·Marty Cagan·Brian Balfour·Geoffrey Moore·April Dunford·Amazon(Working Backwards)의 실제 프레임을 리서치해, 기존 9단계 매트릭스를 진단·개선·확장한 통합본.

---

## 이 매트릭스의 사용법 (한 눈에)

```
  [1층 발굴]  뭘 만들지 고른다      →  후보 아이디어 + 즉시 킬 필터
  [2층 설계]  될 서비스로 빚는다     →  문제→비치헤드→유통→포지셔닝→MVP 순서
  [3층 검증]  될지 채점한다          →  가중치+게이트 채점 (합산 금지)
  [4층 성장]  성공시킨다             →  성장루프·리텐션·PMF·확장
```
각 층은 **아래가 튼튼해야 위가 의미 있음.** ❌ 나온 곳이 급소.

## 🥇 먼저: 프로가 실제로 매기는 가중치 (기존 매트릭스 최대 오류 = 균등배점)

| 축 | 프로 가중치 | 근거 |
|----|:---:|------|
| 시장 크기·성장 + 타이밍 | **30~35%** | Andreessen "시장이 전부" / Gross 타이밍 42% |
| 리텐션·PMF 증거 | **20%** | Sean Ellis 40% 테스트 |
| 유통·성장루프 | **15~20%** | Thiel "유통 실패가 최다 사망원인" |
| 지불 의향 | **10%** | Mom Test "과거 지출이 진실" |
| 팀·창업자 적합(Why you) | **15%** | Gross 팀 32% |
| 해자 | **5% (초기엔 낮게!)** | a16z "해자는 이기고 나서 생김" |

> ⚠️ **합산이 아니라 곱셈/게이트.** 시장·문제·지불의향 중 하나라도 0이면 → 디자인·참신함이 아무리 강해도 못 덮음. 즉시 재고.

# 1층 · 발굴 (DISCOVERY) — 뭘 만들지 고른다

> Paul Graham: **"아이디어는 짜내는 게 아니라 발견하는 것.** 문제를 찾아라, 특히 네가 직접 겪는."

### ⭐ 핵심 순서: 아이디어부터 X → **내 엣지(불공정 우위)부터** 인벤토리
> YC·PG 공통: 아이디어를 먼저 짜고 우위를 찾지 마라. **내 우위를 먼저 목록화하고, 그걸 살리는 아이디어를 발견하라.**
> 4가지를 글로 적어라: ① **엣지**(직업·취미로 남보다 깊이 아는 산업/도구) · ② **프론티어**(유저로서라도 최전선에 있는 급변 분야) · ③ **유통**(공짜로 닿을 청중·리스트·커뮤니티) · ④ **데이터/접근**(남이 못 얻는 데이터·고객접근)
> → 결과물 = "내가 만들 적임자인 *도메인* 목록" (랜덤 아이디어 아님)

### 아이디어 광맥 7 (YC) — 위 도메인 안에서 돌려라
① 내가 잘하는 것 · ② 내가 있었으면 하는 것 · ③ 일하다 직접 만든 것 · ④ 아는 산업의 빈틈 · ⑤ **최근 바뀐 것/새 기술** · ⑥ 크지만 망가진 산업 · ⑦ 사람들에게 직접 물어서

### 수요가 이미 보이는 신호 (Reddit·리뷰·포럼에서 채굴)
- 💰 "이거 돈 내고 쓸래" / 🔧 "우린 수동으로…" "엑셀로 만들어서…" / 😤 "[기존툴] 별로야" (1점 리뷰 마이닝) / 🙏 "이런 툴 없나?"
- **강한 검증** = 같은 불만이 *여러 커뮤니티*에, *상세하게*, *6개월+*, *실패한 시도 언급*과 함께
- 🚩 **스프레드시트가 소프트웨어를 대신하고 있으면** = "여기 안 만들어진 제품 있다"는 최강 신호 (Sequoia)

### 발굴 즉시 4대 킬 필터 (여기서 거른다)
| 필터 | 질문 | 탈락 조건 |
|------|------|----------|
| 🕳️ **타피트** | 친구들이 즉시 좋아함 + B2C + 큰 TAM + 나도 공감? | **= 위험 신호.** YC 지목 무덤 = **행사/이벤트 발견앱·"오늘 뭐하지"·친구약속앱**. 칭찬이 함정 |
| 🎯 **Why You** | 내가 남보다 잘 만들 불공정 우위(도메인·청중·데이터·유통)? | 없으면 후발주자 열위 |
| ⏰ **Why Now** | 3년 전엔 못 했는데 지금 가능해진 변화? | 없으면 왜 아직 안 됐나 |
| 🔥 **진짜 아픔** | 허접한 v1이라도 쓸 만큼 절박한 사람 있나? | 못 답하면 폐기 |

# 2층 · 설계 (DESIGN) — 될 서비스로 빚는다

> 핵심 원칙: **리스크 최우선** (Maurya) — 사업을 죽일 가정을 가장 먼저·가장 싸게 검증. **설계 순서가 곧 성공률.**

### 설계 6단계 순서 (Order of Operations)

**① 문제·고객 못 박기 — Working Backwards (Amazon)**
- 제품 만들기 *전에* **보도자료 + FAQ를 먼저 써라.** "보도자료가 안 써지면 그 제품은 별로다."
- Mom Test 검증: 내 아이디어 말 금지, 상대 삶을 물어라. 진실 = *이미 쓴 시간·돈·임시방편.* "낼래?"(거짓) ❌ → "얼마 쓰고 있나?"(진실) ✅

**② 비치헤드 선택 — Crossing the Chasm (Moore) + Thiel**
- "시장 있나?"(체크) ❌ → **"장악 가능한 가장 좁은 시장을 고른다"(설계)** ✅
- 3조건: *중요할 만큼 크고 / 리드할 만큼 작고 / 내 강점에 맞나.*
- Thiel: **"작게 시작해 독점하라. 항상 너무 작게 시작하는 쪽으로 실수하라."** (페북=하버드부터, 페이팔=파워셀러 수천 명)

**③ 4대 리스크 동시 제거 — Product Discovery (Cagan)**
- **가치**(원하나?) · **사용성**(쓸 수 있나?) · **실현성**(만들 수 있나?) · **사업성**(팔려 돈 되나?)
- "delivery 시작 전에 이 4개를 다 죽여라." 순서: **가치 리스크 먼저.**

**④ 유통을 제품에 처음부터 심기 — Four Fits (Balfour) + Thiel**
- "어떻게 퍼지나?"(체크) ❌ → **"채널을 먼저 고르고, 그 채널 요구사항을 제품에 하드코딩"(설계)** ✅
- **"제품이 채널에 맞춰진다. 채널이 제품에 맞춰지지 않는다."**
- 채널 하나에 올인 (성장의 70%는 단일 채널): 바이럴이면 빠른 가치도달·짧은 사이클 / SEO·UGC면 사용자 콘텐츠 페이지 대량생성 / 페이드면 거래모델
- Thiel: **채널 하나만 작동시켜도 훌륭한 사업. 여러 개 시도하다 하나도 못 잡으면 끝.**

**⑤ 포지셔닝 의도적 설계 — April Dunford**
- 순서: 경쟁 대안(안 하기·인턴 고용 포함) → 차별 속성 → 가치 → 타깃 세그먼트 → **시장 카테고리(맨 마지막에 선택)**
- "카테고리는 주어진 게 아니라 *네가 고르는 레버*." 정면승부 / 니치지배 / 카테고리창조 중 택1.

**⑥ 가장 작은 검증판 — MVP는 학습 도구 (Ries/Blank)**
- MVP ≠ 기능 줄인 v1. = **가장 위험한 가정을 가장 싸게 증명하는 실험.**
- 가정마다 다른 MVP: 수요검증=랜딩/스모크테스트 / 지불의향=선판매 / 가치=컨시어지(수작업) / 흐름=오즈의마법사(수동을 숨김)

# 3층 · 검증 (DIAGNOSIS) — 될지 채점한다

### v2 채점표 (✅강 2 / ⚠️애매 1 / ❌약 0, 가중치·게이트 적용)

| 축 | 핵심 질문 | 가중치 | 게이트 |
|----|----------|:---:|:---:|
| **D1 문제 강도** | 허접한 v1도 쓸 만큼 아픈가? (빈도·긴급·비쌈) | 15% | 🚪0=중단 |
| **D2 시장 크기·성장** | bottom-up으로 충분히 크고 성장 중(~20%/yr)? | 20% | 🚪작으면 중단 |
| **D3 Why Now** | 3렌즈: 기술 준비 / 행동 변화 / 경제성. 지금인 이유? | 15% | — |
| **D4 Why You** | 남보다 잘 만들 불공정 우위? | 12% | — |
| **D5 차별화 + 非타피트** | 다른 각도 + "몇 명이 시도해 죽었고 난 왜 살아남나" | 10% | ⚠️타피트 감점 |
| **D6 유통·성장루프** | 첫 1,000명이 어떻게? 쓰면 다음 1,000명 딸려오나? | 12% | — |
| **D7 지불 의향** | 이미 돈 쓰고 있나? 수익 경로 있나? | 10% | 🚪0=큰 감점 |
| **D8 실현성** | 기간·자원 내 MVP 가능? | 6% | — |
| **D9 해자** | 못 베끼는 것 (초기엔 낮게 — 결과지 투입 아님) | 5% | — |

**판정선 예: 가중 환산 7.5/10 이상이면 진행.** 단, 게이트 하나라도 걸리면 점수 무관 재고.

# 4층 · 성장 (GROWTH) — 성공시킨다

> 여기서 진짜 성패가 갈림. 출시 후 이 신호들로 검증·확장.

| 항목 | 성공 기준 |
|------|----------|
| **리텐션 = 진짜 PMF** | 리텐션 곡선이 *평평해지나*(decay 멈춤). 최고는 "스마일 곡선"(반등) |
| **Sean Ellis 40% 테스트** | "이 서비스 못 쓰면 어떨까?" → **"매우 실망" ≥40%** = PMF |
| **성장 루프 (funnel 아님)** | 산출(유저·콘텐츠)이 다시 입력으로 재투자돼 *복리*로 성장 |
| **비치헤드 독점 → 확장** | 18~24개월 내 한 세그먼트 완전 장악 → 인접 핀 쓰러뜨리기 |
| **채널-모델 적합** | CAC < LTV, 채널 비용구조가 ARPU와 맞나 |
| **지불 전환** | 무료 유저가 실제 돈 내나 (프리미엄 함정 주의) |

# 5층 · VC 투자적합성 (VENTURE FIT) — "될 서비스"를 넘어 "VC가 돈 넣을 서비스"인가

> ⚠️ **핵심 오해 방지: VC 적합 ≠ 좋은 사업.** 1~4층을 다 통과한 "될 서비스"라도 VC엔 안 맞을 수 있음. VC는 *펀드 수익률*을 봄 — 소수 대박이 펀드 전체를 회수하는 구조라, "연 3억 버는 훌륭한 사업"도 거절함(회수 불가). **5층은 게이트가 아니라 필터**: 떨어져도 나쁜 게 아니라 *"VC 말고 정부지원·부트스트랩이 맞는 서비스"*라는 뜻.

### V층 채점표 (✅강 2 / ⚠️애매 1 / ❌약 0)

| 축 | 핵심 질문 | 근거 |
|----|----------|------|
| **V1 벤처 스케일** | 연 30만이 아니라 **연 300억+ 매출** 각이 나오나? (좋은 사업 ≠ 벤처 사업) | VC는 소소한 흑자 사업 거절 |
| **V2 $1B Exit 잠재력** | **TAM이 목표 회수의 20배+** 인가? bottom-up으로 10조 시장 각? | 펀드 회수 구조 (SheetVenture·Pear VC) |
| **V3 팀·트랙레코드** | 실행 이력·도메인 신뢰가 투자자를 설득하나? (Why You보다 넓음) | VC 스코어카드 팀 30% / Gross 팀 32% |
| **V4 자본 효율·다음 라운드** | 얼마 태워서 어디까지? 다음 라운드 각(마일스톤)이 나오나? 번레이트 통제되나 | VC는 "다음 밸류업" 각을 봄 |
| **V5 Why Now 창(窓)** | 이 타이밍 우위가 *경쟁자가 상품화하기 전에* 선점 가능한가 | 레드오션화 속도 (예: SEO강자의 GEO 흡수) |

**판정:** V1·V2가 ❌면 → **VC 트랙 아님**(정부지원·부트스트랩·인수합병 목표로 전환). V1·V2 ✅ + V3~V5 강하면 → **VC 투자 유력.**

### VC 트랙 vs 비(非)VC 트랙 — 어디로 갈지 결정
| | VC가 맞음 | VC 말고 (정부지원·부트스트랩) |
|---|---|---|
| 시장 | 10조+ 급성장 | 작지만 안정적 |
| 성장 | 폭발적 J커브 필요 | 꾸준한 흑자면 OK |
| 자금 | 대규모 태워 선점 | 최소 자본 자립 |
| 예시 | 플랫폼·네트워크효과 | 니치 SaaS·에이전시·로컬 |

## 부록 · 층위 논리 (왜 이 순서인가)
- **1층 발굴**: 엣지·Why Now·진짜 아픔·비(非)타피트로 후보 선별
- **2층 설계**: 문제→비치헤드→4리스크→유통→포지셔닝→MVP (리스크 순)
- **3층 검증 D1~D2 = 시장 게이트**: 여기 막히면 시작 금지 🚪
- **3층 D3~D9**: 타이밍·자격·차별·유통·돈·실행·방어
- **4층 성장**: 리텐션·루프·독점확장으로 실제 성공
- **5층 VC 적합**: 1~4층 통과 후에만 의미. "될 서비스"인 게 확인된 다음 "벤처스케일·투자적합인가"를 물음 (게이트 아닌 필터)
